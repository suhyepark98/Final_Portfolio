ITERATIVE DESIGN

Creating App Design for a Startup

Startup Overview

Statiq

Statiq is a company based in the New Delhi suburb of Gurugram, India, and founded in October 2019.

It allows individuals and buildings to provide EV (electric vehicle) charging points through its app.

Its motivation is to mitigate air pollution from automobile exhaust by encouraging the use of EVs.

Statiq has a multi-revenue model. End users buy a subscription from Statiq to use the network and pay a fee per charging session, which gets split between Statiq and the property owner. This gives landlords who install the stations an incremental revenue boost.

Statiq has installed more than 150 charging stations in Delhi, Bangalore, and Mumbai and the surrounding environments.

Design

We decided to design a mobile app for the startup due to the following reasons:

  • Users will most likely be using the app when they're on the road, looking for EV-charging stations nearby.
  • Users can use the map and navigation features of the app to get to the chosen stations using the phone.

Features:

The app would have a Google Maps-like interface to show various charging options near the user's current location and show the availability, waiting time, distance, rating, types of EV that can be charged, and a navigation feature to bring the user to the station.

Target Users:

Anyone who owns an electric vehicle could benefit from this interface as it helps them locate charging stations faster and make reservations easier.

Lo-Fi Wireframes

We merged selected pages & designs from each set of sketches to build an interface that demonstrates our solution to the startup's goal of reducing pollution by connecting the users & owners of EV charging stations.

The sets of sketches are diverse and creative, and each brings distinct components and ideas, such as map, navigation, profile, reservation, and history pages, as well as sub-components such as spots available and waiting time.
We incorporated all of these critical features into once interface, as shown below.

We categorized each functionality into groups, and designed a navigation bar at the bottom that contains the 3 main feature groups: the Map Page, Profile Page, and History Page.

Studio Critique

Feedback

Interface Design

  • It has good affordances and is easy to navigate.


Confusing Waiting Time

  • Clarification needed on why it is necessary to show waiting time: if there are empty spots, shouldn’t there be no waiting time?


History Page

  • Suggestion to change the graph of monthly spendings to a map of locations/stations visited.


Ambiguity Around Navigation

  • When users reach the end of the reservation, they are provided with a “Swipe Up To Navigate” option.
  • This is a bit confusing because users are unsure whether that refers to a navigation mode or if it just brings them back to the map homepage.

Improvements Made

Fix: Waiting Time

  • Stations that have available spots will show “No Waiting Time.”
  • Only for stations that are currently full will it show an estimated waiting time.


Keep: History Page

  • We decided to stick with showing a monthly cost estimator because we felt like it was a more helpful service for customers.
  • Since we are providing an application that allows people to pay to charge EVs, it might be more helpful to show them how much they are spending as well.
  • While there also might be some users who would like to see which locations they have visited, we felt like it is not as crucial as a function, and could still be done by looking through their own charging history (which lists the stations they charged at).


Fix: Ambiguity Around Navigation

  • We changed the directions from “Swipe Up To Navigate” to “Swipe Up to Enter Navigation Mode” to make it clearer that it is a separate functionality (and not just returning back to homepage).

User Testing

Testing Instructions & Post-Test Questions

Main Task

Make a reservation at Park Station for a Type 2 EV charger at 3:45pm


Sub-Tasks

  1. Start the app using the "Sign In" option.
  2. Choose "Park Station" as your desired EV charging station.
  3. Check if the selected EV Charge Type is "Type 2." If not, select it. Then, reserve.
  4. Reserve the time slot that begins at 3:45 pm & confirm the reservation.
  5. Stop at the page that shows "Reservation Confirmed."


Response Questions

  1. Without clicking on them, what do you think the buttons/icons on the bottom navigation bar do?
  2. On the page showing "Reservation Confirmed," if you were looking to go to the charging station, what action would you take next?
  3. Throughout the process, was it clear & intuitive which buttons/items to click on in order to complete the task and move on to the next step?

Post-Testing Questionnaire

  1. What did you like about this prototype?
  2. Is there anything particularly confusing about this prototype?
  3. What would you change to improve the prototype (making it easier to use/navigate)?
  4. Is there anything you want to talk about that has not been asked?

User Testing Analysis

Task Breakdown

The main task we gave was for the users to complete a reservation procedure, which is the main functionality of our app.

We broke down our main task into sub-tasks, which are actions that users should take on each screen in order to successfully make a reservation

We observed that users typically made little to no errors and have no trouble navigating through the process on their first try. Some of them are confused as to why a payment component of app is needed, but that is due to the fact that the business model of Statiq was not fully communicated to users as background knowledge before they started the test.

Objective Completetion

The fact that most users were able to successfully complete the task on the first try, some even without needing to consult the sub-tasks, shows that the interface is intuitive enough and has good affordances that allows users to know how to go from one step to the next.

This falls largely in line with our expectations based on the similar positive feedbacks received in the group crit studio. Some problems, such as more clarity regarding what the navigation function would look like, was also brought up during user testing.

Potential Changes

Besides changing the navigation text instructions from "Swipe Up to Naviage" to "Swipe Up to Enter Navigation Mode" to make it clear that navigation is a separate functionality that is different than the maps homepage, we could also consider adding an additional screen in our hi-fi mockup to showcase what the specific page would look like to users.

We have also included the names of different functions under the icons in the bottom navigation bar, so the purpose of each icon and where it might take the user would be clear even to people who are using the application for the first time.
